1 Pytest отличается от unittest более простым синтаксисом без наследования от TestCase, мощными фикстурами, встроенной параметризацией через @pytest.mark.parametrize, развитой экосистемой плагинов и использованием стандартного assert с подробным выводом, тогда как unittest требует наследования, методов setUp/tearDown, специальных assert-методов и имеет ограниченные возможности расширения.

2 @pytest.mark.parametrize - это декоратор в pytest, позволяющий запускать один тест с разными наборами входных данных, принимающий имена параметров и список значений, что значительно сокращает дублирование кода и упрощает поддержку тестов.

3 Mock.Mock() создает объект-заглушку для имитации поведения и передачи в тестируемую функцию, а mock.patch() заменяет реальный объект в указанном пространстве имен на mock-объект на время выполнения теста, позволяя подменять импорты и внешние зависимости.

4 Асинхронные функции тестируются с использованием плагина pytest-asyncio, тесты помечаются декоратором @pytest.mark.asyncio, определяются с async def, внутри используются await для вызова асинхронных функций, также можно тестировать конкурентное выполнение и обработку исключений.

5 Coverage.py измеряет процент покрытия кода тестами, отслеживая выполненные строки, при этом 80% считается минимальным уровнем, 90% - хорошим, 95%+ - отличным, но важно покрывать не просто строки, а логические ветви и сценарии использования.

6 Паттерн Repository абстрагирует слой доступа к данным, изолируя бизнес-логику от деталей реализации БД, обеспечивает единообразие операций, упрощает тестирование через подмену репозитория на mock-объекты и используется в системе для централизации всех операций с заказами.

7 Транзакции в SQLAlchemy организуются через session.commit() для сохранения и session.rollback() для отката, явно через session.begin(), а также через контекстный менеджер with session.begin(), и тестирование транзакционности важно для обеспечения атомарности операций и предотвращения неконсистентных данных при ошибках.

8 Проверка отката транзакции выполняется путем создания заказа с некорректными данными, перехвата исключения через pytest.raises, выполнения явного rollback сессии и последующей проверки отсутствия записей в БД через запросы к таблицам orders и order_items.

9 Для эмуляции внешнего API с pytest-httpx используется фикстура httpx_mock, метод add_response для настройки ответа с указанием URL, метода, статуса и тела ответа, после чего вызывается тестируемая функция, проверяется результат и при необходимости анализируется отправленный запрос через get_request.

10 Фикстура с scope определяет область видимости: function создается для каждого теста, class - для класса, module - для модуля, session - для всей сессии, в тестах БД обычно используется scope="function" для создания чистой in-memory SQLite базы с таблицами и автоматического отката после каждого теста, обеспечивая полную изоляцию.
