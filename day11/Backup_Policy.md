## 2. Реализация стратегии защиты данных 3-2-1-1
Для предотвращения потери данных в условиях атак шифровальщиков-вымогателей (Ransomware) и компрометации административных учетных записей применяется отказоустойчивая архитектурная модель:

1. **3 копии данных:** Основная база данных (Production Cluster), инкрементальные снапшоты в облаке AWS S3, полные еженедельные архивы в независимом облаке Azure Blob.
2. **2 разных носителя:** Высокоскоростные блочные NVMe устройства для оперативного восстановления + изолированные объектные хранилища (Object Storage).
3. **1 Offsite (Вне основной площадки):** Данные распределяются и непрерывно реплицируются между тремя независимыми провайдерами: AWS, GCP и Azure.
4. **1 Immutable (Неизменяемая копия):** Использование технологии **S3 Object Lock** в жестком режиме *Compliance Mode* на срок 30 дней. Никакой пользователь, включая скомпрометированный аккаунт root/administrator, не имеет технической возможности удалить или изменить файлы бэкапов до истечения назначенного срока (WORM - Write Once Read Many).
