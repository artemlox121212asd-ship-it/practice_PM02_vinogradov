В ходе выполнения практической работы мною был проанализирован вариант «Гостиница», в рамках которого выделены следующие сущности: Номера, Гости, Бронирования, Услуги и Дополнительные_услуги_в_бронировании. На основе анализа предметной области определены атрибуты для каждой таблицы, первичные ключи и типы связей. В MySQL Workbench построена ER-диаграмма с использованием нотации Crow's Foot, где реализованы связи M:N между Гостями и Номерами через таблицу Бронирования (с дополнительными атрибутами дата заезда и дата выезда), а также M:N между Бронированиями и Услугами через промежуточную таблицу Дополнительные_услуги_в_бронировании. Диаграмма экспортирована в PDF-файл. Сгенерирован SQL-скрипт создания базы данных через Forward Engineer, который успешно выполнен на сервере MySQL с созданием базы данных Variant5_DB и всех таблиц. Подготовлены CSV-файлы с тестовыми данными (Номера, Гости, Бронирования, Услуги, Дополнительные_услуги), которые импортированы через Table Data Import Wizard с разделителем «;» и кодировкой UTF-8, после чего выполнена проверка успешности импорта командами SELECT COUNT(*). Все результаты работы (PDF-диаграмма, SQL-скрипт, CSV-файлы, скриншоты) сохранены в папке Day2 репозитория. В ходе работы возникли небольшие сложности с настройкой связей M:N, требующих создания промежуточных таблиц с составными первичными ключами, а также с сопоставлением столбцов при импорте CSV из-за различий в именованиях атрибутов. Дополнительно создано представление (VIEW) для отображения полной информации о бронировании с данными гостя, номера и списком услуг, что позволяет упростить формирование отчётной документации для администраторов гостиницы. Все файлы зафиксированы в Git-репозитории с соответствующим комментарием, дневник практики заполнен.
