Параметр auto_increment в конфигурации столбца указывает, что значение для этого поля генерируется автоматически при добавлении новой записи (увеличивается на 1), поэтому в интерфейсе такое поле скрывается от пользователя, чтобы он не вводил ID вручную и не нарушал уникальность.

Если в INSERT не указать значения для NOT NULL полей, MySQL выдаст ошибку "Field 'название_поля' doesn't have a default value", и запись не будет добавлена, поэтому в приложении выполняется проверка обязательных полей перед отправкой запроса.

В Tkinter получить выделенную строку из Treeview можно методом selection(), который возвращает список идентификаторов выбранных элементов, затем через item() извлекаются значения: selected = tree.selection(); values = tree.item(selected[0])['values'].

Мы не даём пользователю редактировать поле с первичным ключом, потому что это уникальный идентификатор записи, который используется для связей с другими таблицами через внешние ключи, и его изменение нарушит целостность данных и ссылочные связи.

Защита от SQL-инъекций реализуется использованием параметризованных запросов с плейсхолдерами %s, где значения передаются отдельным кортежем в метод execute() — это гарантирует, что пользовательский ввод обрабатывается как данные, а не как исполняемый SQL-код.
