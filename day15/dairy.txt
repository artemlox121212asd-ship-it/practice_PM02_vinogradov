Разработан комплексный набор тестов для сервиса обработки платежей PaymentService с использованием pytest и unittest.mock. Протестирована логика повторных попыток при ошибках BankApiClient: при TimeoutError сервис выполняет 3 попытки, затем выбрасывает PaymentFailedException с сообщением "Bank unavailable". Проверены успешный сценарий оплаты (вызов charge один раз с правильной суммой, логирование успеха), сценарий с другими исключениями (ValueError, ConnectionError) без повторных попыток, а также корректная обработка частичных отказов (один успешный вызов из трёх). Написаны параметризованные тесты для валидации суммы платежа (отрицательная, нулевая, слишком большая) с ожидаемыми ValidationError. Реализованы тесты для проверки логирования (mock logger с assert_called_with), тест производительности с 1000 одновременных запросов через ThreadPool и тест идемпотентности с одинаковым idempotency_key. Настроены фикстуры в conftest.py (mock_client, payment_service с инжекцией моков). Все тесты успешно проходят, покрытие сервисного слоя составляет 94%. Оформлены скриншоты выполнения с зелёными галочками, отчёт сохранён в репозиторий.
