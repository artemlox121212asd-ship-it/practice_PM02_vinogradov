pytest проще и мощнее unittest: меньше кода, фикстуры, параметризация, удобные assert'ы.

Фикстуры — функции, подготавливающие ресурсы для тестов (БД, клиенты), обеспечивают изоляцию и переиспользование.

Мокать HTTP — через pytest-httpx, responses или unittest.mock.patch для requests/httpx.

@pytest.mark.parametrize — запускает один тест с разными наборами данных.

Проверить исключение — with pytest.raises(Ошибка, match="текст"): func().

Mock() создаёт заглушку, patch() подменяет существующий объект на время теста.

Асинхронные тесты — через pytest-asyncio с маркером @pytest.mark.asyncio.

Фабрика фикстур — функция, создающая параметризуемые фикстуры для разных вариантов ресурсов.

Проверить вызовы — через assert mock.method.call_count == N и assert_has_calls().

coverage.py — считает процент покрытия кода тестами; хороший порог — 80-90%.
