1.Сессия: создать — session['user_id'] = 1, проверить — if 'user_id' in session

2.Хеширование паролей: защищает данные при утечке БД, bcrypt — hashpw() для создания, checkpw() для проверки

3.@login_required: проверяет авторизацию перед выполнением функции, иначе редирект на логин

4.Капча: сложение двух случайных чисел (можно обойти, если знать алгоритм)

5.Права: админ — всё (пользователи, бронирования), работник — только просмотр и изменение статуса бронирований

6.Защита маршрута: декоратор @admin_required проверяет session['role'] == 'admin'

7.Запуск на новом ПК: pip install -r requirements.txt → python create_users.py → python app.py

8.requirements.txt: список зависимостей проекта, создать через pip freeze > requirements.txt
