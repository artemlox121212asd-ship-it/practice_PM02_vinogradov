При первом запуске скрипта возникла ошибка StopIteration из-за того, что цикл выполнялся на одну итерацию больше количества партий в генераторе. С помощью вставки breakpoint() и использования команд pdb (p, pp, n, s) были локализованы неверные значения переменных: в функции process_data_value оператор битового сдвига >> использовался вместо умножения, из-за чего результат вычислений был неверным. Для отладки рекурсивной ошибки были установлены условные точки останова, например, остановка при depth > 90 в функции deep_calc, которые сохранены в файл .pdbrc. Анализ памяти с помощью tracemalloc выявил три утечки: бесконечное накопление данных в замыкании create_processor_with_leak (7840 KiB), неограниченный рост глобального кеша в рекурсивной функции (1000 KiB) и накопление объектов в стеке рекурсии (512 KiB). Исправления включали замену range(batch_count + 1) на range(batch_count), замену оператора >> на *, добавление условий выхода из рекурсии по глубине и значению, а также ограничение размера кешей с удалением старых записей. После всех исправлений все 12 тестов pytest успешно прошли, повторный запуск tracemalloc подтвердил отсутствие утечек памяти с ростом менее 128 KiB. Оформлен подробный отчет с root cause analysis для каждой ошибки, описаны причины возникновения и способы предотвращения в будущем. В результате программа стала полностью работоспособной, стабильной и эффективно использующей память.
