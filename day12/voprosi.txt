Стек вызовов (call stack) — это структура данных, хранящая последовательность вызовов функций, которые привели к текущей точке выполнения программы. В момент ошибки его можно посмотреть через вывод трассировки (traceback) в консоли, используя модуль traceback (traceback.print_exc()), в pdb командой where (или w), в отладчике IDE через панель Debug. В момент необработанного исключения интерпретатор автоматически печатает стек в stderr.

В pdb команда p (print) выводит значение выражения в удобочитаемом формате (как repr), а pp (pretty-print) использует модуль pprint для красивого форматирования сложных структур (словарей, списков) с отступами и переносами строк, что делает данные более наглядными при больших объёмах.

В pdb перейти из дочерней функции обратно в родительскую (на уровень выше) можно командой u (up), которая перемещает текущий фрейм на один уровень вверх по стеку вызовов, где можно осмотреть локальные переменные родительской функции. Для возврата на уровень ниже используется команда d (down).

Модуль tracemalloc предназначен для отслеживания выделения памяти и поиска утечек. Для получения топ-10 строк кода по потреблению памяти нужно запустить tracemalloc.start(), выполнить код, затем вызвать snapshot = tracemalloc.take_snapshot(); top_stats = snapshot.statistics('lineno')[:10] и вывести их, что покажет файл, строку и объём выделенной памяти в байтах.

Запустить программу в режиме отладки сразу, не вставляя breakpoint() в код, можно командой python -m pdb имя_файла.py — это запускает интерпретатор с модулем pdb, который останавливается на первой строке; также можно использовать python -m debugpy --listen ... для внешних отладчиков, или в IDE (PyCharm, VSCode) установить точку останова и запустить режим Debug.

Использование пустого except: считается антипаттерном, потому что оно перехватывает все исключения, включая SystemExit, KeyboardInterrupt и ошибки, которые не предполагалось обрабатывать, что скрывает критические ошибки, затрудняет отладку, маскирует баги и нарушает принцип "fail-fast". Вместо этого следует перехватывать конкретные типы исключений (except ValueError: или except Exception as e: с логированием, но не KeyboardInterrupt)
